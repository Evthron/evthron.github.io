實習學會的

2026-07-07
  • 大學真可怕。沒有薪水、沒有規律的時間表、自己逼自己工作，還有目標不明。
  • 換個角度想，能畢業就確保了基礎的工作能力，僱主沒有任何理由聘用大學生以外的人。

軟件開發

  • 在實際有人維護的代碼庫中的工作經驗是很寶貴的。只有自己工作的 git repository 永遠也不需要用到 git branch 和 git merge。
  • 工作時間越是零碎，就越是容易在不清楚真正需求的情況下趕工，造出其實不需要的東西，浪費時間。
  • 不得不承認， OOP 和各種流行框架在團體開發的時候是必不可少的，能讓新人在不瞭解系統全貌的情況下靠約定和 API 工作。代價是拖累開發速度和增加系統的複雜度。單獨開發的時候完全沒有必要使用這些方法。

溝通成本

  • 雖然工作內容都是會向新手解釋一遍的，但因為溝通成本高和專家忘記了新手痛苦的關係，往往還是要自己研讀代碼，還有向同事了解工作的目的和隱藏的代碼規範。
  • 即時通訊是最有效率的交流方式，前提是依然需要同一個物理工作空間工作，這樣就不存在之前説過的信任問題。打斷別人寫代碼的思路是很損害工作效率的。

AI

  • 只要有 AI 魔法，就算完全不懂新語言語法的新人也能馬上開始工作，一個人頂兩個人用。
  • 完成任務最重要，很容易依賴 AI，但工作完成了之後就沒有理由再碰，弄懂自己在做什麼變成奢侈的事。過程中要儘可能爭取時間理解一點工作背景和周遭的代碼。
  • 如果想要讓 AI 工具在工作中落地，寧可放棄性能，也必須要有便宜可驗證的輸出結果。如果只用關鍵詞和傳統機器學習模型，又會覺得落後，需要權衡。
  • 不會被取代的。AI 只是個裁員的藉口。軟件開發是投資，無法創造直接利潤，賺不到錢的時候自然會被炒掉。AI 完成的工作越多，就會產生更多維護的工作。不清理，代碼庫就會逐漸腐爛。
  • 真正的危機來自學習速度的提升導致的知識貶值，而不是完成工作的速度本身。
